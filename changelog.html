Release notes

Changelog

Release status and improvements for DHC-6 Trainer mobile, desktop, live trainer, and website.

Latest

Desktop v1.7.4 — Full 3D Systems Lab + asset wiring

Systems Lab upgraded to nine system groups: Engine, Propeller, Fuel, Electrical, Hydraulics, Fire, Landing Gear, Bleed Air/Pneumatics, and Aircraft Variants. GLB path matching corrected to align with the 19 real models in core-res (PT6A-27 cutaway, Hartzell propeller, fuel system, hydraulic pack, DHC-6 aircraft variants, and more). The jMonkeyEngine 3D viewer now loads real models when built from the full project. New camera presets per system group from DesktopSystemModelMap. Aircraft-variant canvas height raised for the wide fuselage models.

Desktop v1.7.3 — Proper 3D engine viewer

jMonkeyEngine 3D viewer enabled

Added the five jMonkeyEngine JME3 3.6.1-stable dependencies that were missing from build.gradle — the root cause of the 3D viewer not rendering. The viewer now has an orbit/arcball camera (drag to rotate, scroll to zoom), a 4-light premium setup, Bloom + FXAA post-processing, auto-rotate when idle, and a PT6A-27 procedural fallback engine built from JME3 cylinders. Engine group gets a tabbed view: 3D Model / Cross-section diagram.

Desktop v1.7.2 — PT6A-27 cross-section viewer

Interactive engine diagram

Added Pt6aEngineViewer.kt — an interactive PT6A-27 engine cross-section built with standard Compose layout. Eight colour-coded sections with click-to-highlight and a description panel sourced from the FCTM. Replaces the Systems Lab placeholder for the Engine group.

Desktop v1.7.1 — Logbook + score persistence

Drill scores saved locally

Added DesktopProgressStore.kt using java.util.prefs.Preferences. MCC Callout and Drill sessions now record to a local logbook. The MCC home card shows real session counts and best-score %. The Logbook screen replaced with a live history: summary tiles, focus-areas debrief (lowest average by procedure title), and a scrollable recent-attempts list with date, time, and score.

Desktop v1.7.0 — Logo + drill graphics

TJ Aeronautical logo surfaced in the UI

Navigation-rail brand tile replaced the placeholder “6” with the real TJ Aeronautical Twin Otter logo. MCC Callout home hero and session view have graphics. Drill (A/B/C/D) screen has a logo banner. Severity annunciator (warning/caution/default) displayed beside the “PM CALLS” header, keyed to the drill category.

Live Trainer — full redesign

In-browser trainer rebuilt in the new design system

The live trainer at live.html is rebuilt using site-redesign.css. Seven modules: Procedures, QRH, scored Flashcards, Performance, scored Cockpit Drill, Logbook, and an interactive 3D PT6A-27 cross-section. Every session is saved to a local logbook with best scores and a focus-areas debrief. 25 flashcards covering PT6A-27 limits, V-speeds, and fuel/oil/hydraulic/electrical systems. Three QRH items: Engine Fire In-Flight, Engine Fire On Ground, and Generator Fail. Performance flags MTOW/MLW and tailwind exceedances. All content is grounded in the DHC-6 AFM/POH, operator QRH, and FCTM; data stays in the browser.

Website

Site redesign: site-redesign.css

All main pages now use a unified design system: navy cockpit-dark background, blue/cyan accent palette, glass cards, smooth responsive grid. Screenshots and product images updated. Desktop page corrected to show current version. Changelog migrated to the new design system.
